# 06 · 唤醒与收尾

## 上下文（给 Trae）
05 完成：三屏 + 语音 + 切场检查都齐了。最后接「语音口令唤醒」+ 打磨可装、可离线。

## 要做的
- **读 URL 参数**：打开时若地址带 `?log=1`（例如 `index.html?log=1`），自动进记录页并**自动启动语音识别**（复用 04 的逻辑，相当于自动按下「按住说话」）。如果设备不支持语音，进了记录页但不报错。
- **项目根 README**：写一段「配 Google 助手例程」的步骤，让用户配：说「嘿 Google，记忘事」-> 打开 `https://你的地址/index.html?log=1`。（地址留占位让用户填。）
- **检查 PWA 可装**：`manifest.json` 的 `display=standalone`、有 `icons`、有 `start_url`、`name`/`short_name` 齐全。
- **离线**：确认 `sw.js` 把 `index.html / main.js / db.js / style.css` 都缓存了，断网能开。
- **整体自测**：装桌面 -> 点图标开 -> 记一笔（语音）-> 看板能看到 -> 断网再开还能用。

## 约束
不要加后端、不要加账号。这条只做唤醒集成和打磨。

## 验收标准
1. 用 `index.html?log=1` 打开，自动跳到记录页并开始录音（看浏览器麦克风权限提示）。
2. 离线（断网）打开页面能用，数据还在。
3. 桌面图标打开是全屏、无浏览器地址栏（standalone 生效）。
4. README 里有清晰的「嘿 Google 记忘事」配置步骤。
