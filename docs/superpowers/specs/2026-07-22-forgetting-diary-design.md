# 忘事日记 (ForgetLog) - v0 设计文档

**日期**: 2026-07-22
**状态**: 已确认设计，待用户评审
**作者**: 用户 + AI 协作（火山引擎 / Trae 类 AI IDE）

---

## 1. 背景与问题

用户订阅了 AI agent plan，想做"自己天天能用 + 能给别人看"的小应用但缺灵感。经梳理确定方向：

- 用户在药品生产领域做批记录编写与实验（有过团队交付自动填写模板的经验）。
- 自我观察：**注意力被短视频（抖音）和番茄小说带偏，导致经常忘事**（忘带东西如蛋糕落在车上、忘拿快递、忘要办的事）。
- 忘事形状：实物（切场那一刻丢）+ 任务（到点没想起来），约各半。
- 根因判断：用户选择优先解决"整体注意力被磨薄"（机制 B / D），以"忘事"作为抓手与信号。

**核心洞察**：市面上管注意力的工具量的都是"时间"（刷了几分钟），而用户真实痛是"忘事"。把"我又忘了"当信号、而非"屏幕时间"当信号——这个角度基本没人做。

## 2. 目标与非目标

**目标（v0）**
- 自己每天能用、低摩擦地记录"忘事事件"。
- 用一周真实使用数据，验证"记忘事"这个习惯是否真有改善。
- 据使用感受决定下一步投入。

**非目标（v0，明确不做）**
- app 拦截 / 屏幕时间限制（交给 Android 系统自带数字健康）。
- 账号、云同步、分享。
- 自动活动检测、提醒、磁贴 / 双击唤醒。

## 3. v0 范围（三屏）

### 屏 1 · 主页
- 大按钮"记一笔忘事" -> 进入记录。
- 第二按钮"切场检查"。

### 屏 2 · 记录
- 按住语音按钮说话 -> 自动转文字（优先）；或手动打字。
- 场景标签 chip（多选，可空）：切换场景 / 到家 / 下车 / 出门 / 刚刷完手机。
- 自动记时间；保存回主页。

### 屏 3 · 看板
- 这周记了几笔。
- 按场景分布、按星期几分布（纯数字 + 简单条，不搞花图表）。
- 最近 10 条记录列表，可删。

## 4. 架构与技术栈

- **形态**：PWA（网页 app，加到桌面当 app 用）。
- **存储**：IndexedDB，本地，无后端。
- **语音**：Web Speech API（SpeechRecognition），免费、离线、无 key。
- **唤醒**：
  - 桌面图标一点即开。
  - "嘿 Google，记忘事"语音口令 -> 打开 PWA（可带 URL 参数自动进入记录页并启动语音）。
- **构建**：用 AI IDE（火山引擎 / Trae）编写；尽量无构建步骤（vanilla JS + manifest + service worker），保持极简，便于非程序员维护。

## 5. 数据模型

单一对象存储 `entries`：
- `id`：字符串（时间戳或 uuid）。
- `created_at`：ISO datetime。
- `text`：忘了啥（语音转写或打字）。
- `tags`：字符串数组（场景标签，可空）。
- 派生（统计用，不入库）：`weekday`、`hour`。

## 6. "走路 / 开车禁抖音番茄" 的处理

**不在 v0 内。** 原因：PWA 无法拦截别的 app，后台活动检测也需原生。

**当前方案**：用 Android 系统自带：
- 数字健康 -> 专注模式，锁"抖音""番茄小说"。
- 驾驶模式 / Heads Up（走路提醒）。

先用系统自带白嫖一周，验证是否有效；若有效已白赚，若仍控不住再评估做成 v2 原生模块。

## 7. 成功标准（v0）

- 能装到手机桌面，1 周内真实记录忘事事件。
- 单次记录 < 10 秒（语音）。
- 一周后看板有真实数据，据此决定下一步。
